# 🩺 Health Prediction Application

An AI-powered health risk prediction system built with Streamlit, scikit-learn, and SQLite. The application collects patient blood test data, runs it through a trained `RandomForestClassifier`, and automatically generates a health risk category and clinical-style remark — all backed by full CRUD patient record management and a live analytics dashboard.

---

## 📋 Project Overview

Healthcare providers and clinics often need a fast way to triage routine blood test results and flag patients who may need follow-up. This application demonstrates an end-to-end mini health-tech system:

- Patients are registered with basic demographic and contact information.
- Three core blood markers — **Glucose**, **Haemoglobin**, and **Cholesterol** — are recorded.
- A locally trained machine learning model classifies each patient into one of four risk categories and generates a human-readable remark.
- All records are persisted in a local SQLite database and can be created, viewed, updated, and deleted through a clean Streamlit interface.

No external LLM or AI API (ChatGPT, Gemini, Claude, etc.) is used anywhere in the prediction pipeline — the "AI" in this project is a genuinely trained `RandomForestClassifier` running entirely offline on a synthetic dataset generated by this codebase.

---

## ✨ Features

### Core Functionality
- **Full CRUD** — Create, Read, Update, and Delete patient records.
- **Automatic AI Prediction** — every create/update re-runs the ML model and refreshes the remark.
- **Persistent Storage** — SQLite database survives application restarts.
- **Data Validation** — required fields, regex email validation, future-date rejection for DOB, and clinically-bounded numeric ranges for all blood markers.

### Dashboard
- Live metric cards: Total Patients, Healthy, Prediabetes Risk, Anemia Risk, High Cholesterol Risk.
- Interactive Plotly donut chart of prediction distribution.
- Recent patients list with color-coded risk badges.

### Patient Management
- Searchable, sortable patient table.
- Inline edit form with automatic re-prediction on save.
- Delete with an explicit confirmation checkbox to prevent accidental data loss.

### Machine Learning
- Realistic synthetic dataset generator (1,200 records across 4 balanced classes with injected measurement noise).
- `RandomForestClassifier` trained with class balancing and evaluated via a held-out test split.
- Model persisted with **Joblib** and loaded once per process for fast inference.
- Probability/confidence breakdown available for every prediction.

## 🏗️ Architecture

```
health_prediction_app/
│
├── app.py                          # Streamlit UI: dashboard, add/manage patient pages
├── database.py                     # SQLite connection handling + CRUD functions
├── model.py                        # Loads trained model, exposes predict_health_risk()
├── validation.py                   # Field-level and form-level validation utilities
├── train_model.py                  # Synthetic data generation + RandomForest training
├── requirements.txt                # Python dependencies
├── README.md                       # This file
│
├── models/
│   └── health_model.pkl            # Serialized model bundle (model + metadata)
│
└── data/
    ├── synthetic_health_data.csv   # Generated training dataset (1,200 rows)
    └── health_records.db           # SQLite database (created on first run)
```

### Module responsibilities

| Module | Responsibility |
|---|---|
| `validation.py` | Pure functions, no dependencies on UI/DB/ML. Validates name, email, DOB, and each blood marker. |
| `database.py` | All SQL lives here. Exposes `create_patient`, `get_all_patients`, `get_patient_by_id`, `update_patient`, `delete_patient`, `get_prediction_counts`. |
| `train_model.py` | Generates the synthetic dataset, trains the `RandomForestClassifier`, evaluates it, and saves the model bundle via Joblib. Runnable standalone. |
| `model.py` | Thin inference wrapper. Loads the saved model bundle (training automatically on first use if missing) and exposes `predict_health_risk()` / `predict_with_probabilities()`. |
| `app.py` | Streamlit presentation layer. Wires together validation, database, and model modules into three pages: Dashboard, Add Patient, Manage Patients. |

This separation means the ML and persistence logic can be tested or reused independently of the Streamlit UI (e.g. from a script or a future API layer).

## ⚙️ Installation Steps

**Prerequisites:** Python 3.11 or higher.

1. Clone or download the project folder:
   ```bash
   cd health_prediction_app
   ```

2. (Recommended) Create and activate a virtual environment:
   ```bash
   python3 -m venv venv
   source venv/bin/activate      # On Windows: venv\Scripts\activate
   ```

3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

---

## ▶️ Running the Application

1. **Train the model** (only needed once — the app will also auto-train on first launch if the model file is missing):
   ```bash
   python train_model.py
   ```
   This generates `data/synthetic_health_data.csv` and `models/health_model.pkl`, and prints accuracy/classification metrics to the console.

2. **Launch the Streamlit app:**
   ```bash
   streamlit run app.py
   ```

3. Open the URL shown in the terminal (typically `http://localhost:8501`) in your browser.

4. Use the sidebar to navigate between **Dashboard**, **Add Patient**, and **Manage Patients**.

## 🤖 Model Training Details

The model is trained entirely on a **synthetic dataset generated within this project** — no external data source or LLM is used.

- **Generation strategy:** Each of the four target classes (`Healthy`, `Prediabetes Risk`, `Anemia Risk`, `High Cholesterol Risk`) is sampled from class-specific Gaussian distributions over Glucose, Haemoglobin, and Cholesterol, loosely informed by common clinical reference ranges. A small fraction (~4%) of records receive extra noise to simulate real-world borderline cases and avoid an unrealistically clean decision boundary.
- **Volume:** 1,200 records (300 per class).
- **Algorithm:** `sklearn.ensemble.RandomForestClassifier` with 200 estimators, balanced class weights, and a constrained max depth to reduce overfitting risk.
- **Evaluation:** 80/20 stratified train/test split; typical held-out accuracy is in the high-90s% range, with feature importances roughly evenly distributed across the three markers (as expected, since each class is primarily driven by one distinguishing feature).
- **Persistence:** The trained model, expected feature column order, class labels, and the remarks lookup table are all bundled together and saved with `joblib.dump()` so inference code never has to guess input shape or label meaning.

To regenerate the dataset and retrain from scratch at any time:
```bash
python train_model.py
```
